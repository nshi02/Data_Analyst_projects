# Проекты

- [Анализ работы сервиса бронирования экскурсий на основе пользовательских взаимодействий](./User%20activity)
  - **Что делалось:**
    - Анализ эффективности использования фильтров и работы сервиса
    - Построение воронок событий
    - Поиск мест для улучшения
  - **Стек:** Python, Pandas, NumPy, Matplotlib, Seaborn, Jupyter Notebook
  - **Результаты:**
    - Сервис в основном используется для ознакомления с главной страницей города. Процент перехода к фильтрации/поиску событий в среднем составил 5%.
    - Корреляция между спросом и предложением составила 87%, что указывает на хорошую работу сервиса.
    - Выявлено ~30 направлений, пользующихся спросом, но предлагающих только 4-7 релевантных предложений.

- [Анализ пользовательской активности сервиса бронирования для путешествий](./Booking%20Data)
  - **Что делалось:**
    - EDA
    - Проверка гипотез
    - Поиск мест для улучшения
  - **Стек:** Python, Pandas, NumPy, Matplotlib, Seaborn, Jupyter Notebook
  - **Результаты:**
    - Основное прибыльное направление — авиабилеты, на них приходится около 80% всех заказов.
    - Средняя стоимость заказа ~15 000 рублей (~153 евро). Большинство заказов до 9 000 рублей; распределение несимметричное с длинным левым хвостом.
    - По городам России лидируют Москва, Сочи, Екатеринбург, Минеральные Воды, затем Санкт-Петербург.
    - Местоположение клиента влияет на частоту и вероятность покупки. Необходимы кэшбеки, скидки и персонализированные предложения для повышения вовлеченности.
